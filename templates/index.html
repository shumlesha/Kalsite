ФЗ: функциональная зависимость.

НФБК: нормальная Форма Бойса-Кодда. Отношение находится в нормальной форме Бойса-Кодда (НФБК), если каждый детерминант атрибутов отношения является возможным ключом отношения.

Детерминант: Если А->B (A функционально определяет B) и не зависит функционально от любого подмножества А, то А представляет собой детерминант B.

Возможный ключ, Ключ-кандидат, Потенциальный ключ: Атрибут (группа атрибутов), функционально определяющее отношение.

Суперключ: атрибут или их множество, единственным образом идентифицирующее кортеж отношения.

Потенциальный ключ: суперключ, никакое подмножество которого не является суперключом.

Составной ключ: ключ, состоящий из нескольких атрибутов.

Первичный ключ: потенциальный ключ, выбранный для уникальной идентификации кортежей отношения.

Альтернативный ключ: потенциальный ключ, не выбранный в качестве первичного.

Суррогатный ключ: искусственный атрибут, не имеющий связей с реальными характеристиками явлений ПрО, вводимый в схему отношения исключительно для организации связи кортежей.

Суррогатный первичный ключ: первичный ключ, уникальные значения которого генерируются СУБД.

Внешний ключ: атрибут отношения или множество атрибутов, соответствующее потенциальному ключу некоторого отношения и являющееся его подмножеством. "внешний ключ должен являться подмножеством значений атрибутов на которые он ссылается".

Триггер: программа на языке программирования сервера, которая автоматически выполняется СУБД в момент наступления определенного события.

Курсор: инструмент для построчного сканирования результирующих таблиц с данными.

unknown: третье значение логического типа, обладающее свойствами: (для удобства запоминания можно представлять unkn0wn = 0.5, true = 1, false = 0).

Кортеж: набор именованных значений заданных типов (это множество пар «имя атрибута, значение», которое содержит одно вхождение каждого имени атрибута, принадлежащего схеме отношения).

Схема отношения: это именованное множество пар «имя атрибута, имя домена».

Схема БД: это набор именованных схем отношений.

Мощность множества: степень схемы отношения, которая определяет количество атрибутов в схеме.

Реляционная база данных: это набор отношений, имена которых совпадают с именами схем отношений в схеме БД.

Отношение: множество кортежей, соответствующих одной схеме отношения.

Домен: множество допустимых значений данного типа.

Атрибут: именованный домен, представляющий семантически значимые объекты.

Отношение: множество кортежей, соответствующих одной схеме отношения.

Схема отношения: это именованное множество пар «имя атрибута, имя домена».

Схема БД: это набор именованных схем отношений.

Мощность множества: степень схемы отношения.

Представление: виртуальное отношение, кортежи которого не хранятся на диске, а воспроизводятся на основании базовых отношений, реально существующих в БД.

Аксиомы Армстронга: набор правил вывода, используемых для вывода всех функциональных зависимостей реляционной базы данных.

Универсальное отношение: отношение, в которое включаются все представляющие интерес атрибуты ПрО и которое может таким образом содержать все данные, предполагаемые к хранению в БД. Для малых БД (включающих не более 15-20 атрибутов) универсальное отношение может использоваться в качестве отправной точки при проектировании РБД.
